# Launch Baseline

## Что должно быть готово перед публичным показом

- Главная загружается без server error и выглядит цельно на desktop и mobile.
- Каталог открывается быстро, фильтры на телефоне не конфликтуют с картой.
- На карточке объекта сразу видны цена, расстояние до воды, формат отдыха и основной CTA.
- `USER` может зарегистрироваться, войти, сохранить объект, открыть избранное и отправить отзыв.
- `OWNER` может войти, обновить карточку, увидеть требования к публикации и отправить объект на модерацию.
- `ADMIN` может открыть объект из очереди, опубликовать/отклонить карточку и модерировать отзывы.

## Production smoke-pass

### Публичная часть

1. Открыть `/`
2. Перейти в `/catalog`
3. Открыть mobile filters
4. Убедиться, что карта не мешает фильтрам
5. Открыть одну карточку объекта
6. Проверить:
   - цена
   - расстояние до воды
   - фото
   - быстрый контакт
   - заявка
   - отзывы

### USER

1. Зарегистрировать новый аккаунт
2. Войти
3. Добавить объект в избранное
4. Проверить `/favorites`
5. Оставить отзыв
6. Проверить `/account`

### OWNER

1. Войти как `owner@alakol.kz`
2. Открыть `/owner`
3. Зайти в объект
4. Изменить карточку
5. Убедиться, что виден блок требований к публикации
6. Отправить карточку на модерацию

### ADMIN

1. Войти как `admin@alakol.kz`
2. Открыть `/admin`
3. Открыть pending-объект
4. Опубликовать или отклонить
5. Проверить модерацию отзывов

## Что считаем нормальным UX

- Публичные страницы ощущаются быстрыми без 5–10 секунд ожидания.
- При клике у кнопок есть визуальный отклик.
- На мобильном первый полезный сценарий читается без визуального шума.
- Пустые или слабые карточки не должны выглядеть как “сломанный прод”.
